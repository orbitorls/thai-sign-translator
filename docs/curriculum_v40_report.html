🐛 Root Cause ที่แก้ (v40 = version ที่ 40 หลังพยายาม)

# ─── BUG (v1–v39): smoke_args.epochs ถูก hardcode เป็น 1 เสมอ ─── # เมื่อ seed อยู่ที่ epoch=76 และ target=1 → trainer ไม่เดินเลย (76 > 1) # → noop-resume guard fire → RuntimeError: zero new optimizer steps smoke_args.epochs = max(1, min(int(getattr(kaggle_args, "epochs", 1) or 1), 1)) # ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ # คณิตศาสตร์นี้ให้ผลเป็น 1 เสมอ ไม่ว่า epochs จะเป็นเท่าไร # ─── FIX (v40): ให้ max_train_steps เป็นตัวหยุดแทน epoch limit ─── smoke_args.epochs = 100000 # max_train_steps=20 (relative new steps) ควบคุมการหยุด smoke แทน # epoch=100000 ไม่มีทางถึง → training หยุดเมื่อครบ 20 steps จริงๆ # ✓ v40 smoke ผ่าน → Phase 1 รันได้ 3,725 steps → Phase 2 +375 steps → chrF 96.77
ผลกระทบ: bug นี้ทำให้ kernel ล้มเหลวในเวลา <3 นาที ทุกครั้ง ตั้งแต่ v1–v39 (38 versions) โดยไม่มีใครสังเกตเห็น เพราะ error message ชี้ไปที่ noop-resume guard ไม่ใช่ที่ hardcoded epochs
