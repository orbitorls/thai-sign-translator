01 บทนำและวัตถุประสงค์

รายงานนี้สรุปผลลัพธ์ของ Curriculum Training v40 สำหรับโมเดล PoseToTextT5 ซึ่งเป็นระบบแปลภาษามือไทย (Thai Sign Language: TSL) เป็นข้อความภาษาไทย โดยใช้ข้อมูล pose keypoints จากวิดีโอเป็น input และ mT5-small เป็น text decoder

โปรเจกต์นี้แก้ปัญหาหลักสองข้อ: (1) โมเดลเดิม (incumbent) มี chrF 86.95 ซึ่งยังมีที่สำหรับปรับปรุง และ (2) กระบวนการ train บน cloud ล้มเหลวทุกครั้งมานาน 38 versions เนื่องจาก bug ที่ซ่อนอยู่ใน smoke training

ผลลัพธ์: v40 แก้ bug ได้สำเร็จ ทำให้ pipeline รันจนจบได้เป็นครั้งแรก และได้โมเดลที่ chrF 96.77 (+9.82 จากเดิม) ซึ่งถูก promote เข้าสู่ production ทันที

02 Methodology — แนวทางและวิธีการ

2.1 Curriculum Learning

Curriculum Learning คือแนวทางที่ให้โมเดลเรียนจากข้อมูลทั่วไปก่อน แล้วค่อย finetune กับข้อมูลเป้าหมาย ซึ่งช่วยให้โมเดลสร้าง representation กว้างก่อน แล้วจึงเฉพาะเจาะจงในขั้นสุดท้าย

Phase 1 — Pretrain (ข้อมูลกว้าง)
Train บนทุก dataset รวม 1,913 ตัวอย่าง
เป้าหมาย: ให้ pose encoder เห็น pose หลากหลาย
Early-stop: val_loss · patience=8
lr = 5×10⁻⁵
Phase 2 — Finetune (ข้อมูลเป้าหมาย)
Train เฉพาะ TSL-51 (227 ตัวอย่าง)
เป้าหมาย: ให้โมเดลคมชัดบน vocabulary เป้าหมาย
Early-stop: val_chrf · patience=10
lr = 5×10⁻⁶ (ต่ำกว่า 10×)

2.2 Source Balancing & Focus Tokens

Phase 1 ใช้ auto source balancing เพื่อป้องกันไม่ให้ YouTube dataset (85% ของข้อมูล) dominate การเรียนรู้ นอกจากนี้ยังใช้ focus token multiplier เพื่อ upweight ตัวอย่างที่มีคำสำคัญ เช่น ฉัน, คุณ, แม่, พี่ (rare kinship pronouns)

2.3 Evaluation Protocol

ใช้โปรโตคอลเดียวกันกับ incumbent ทุกอย่าง: n=25, seed=42, data=tsl51_v3, split=manifest, beam_size=5 ทำให้การเปรียบเทียบ valid โดยไม่มี confounding variables

Promotion Guardrail: โมเดลใหม่จะถูก promote เข้า production เฉพาะเมื่อชนะ incumbent ทุก metric พร้อมกัน — ถ้าแพ้แม้แต่ metric เดียว จะ reject โดยอัตโนมัติ ป้องกัน live regression

Guardrail Logic

ระบบ guardrail ทำงานใน scripts/promote_pose_t5_export.py:

  1. ตรวจ manifest_quality.passed
  2. ตรวจ chrF ≥ 80.0 AND candidate > incumbent
  3. ตรวจ exact_match_pct ≥ 60.0
  4. ถ้าทุกข้อผ่าน → copy ไฟล์ทั้งหมดเข้า stable dir
  5. ถ้าข้อใดข้อหนึ่ง fail → reject, stable dir ไม่เปลี่ยน
Safety guarantee: worst case = reject → incumbent เดิมยัง serve อยู่ ไม่มีทาง live regression

13 Root Cause Fix (v1–v39 → v40)

Root cause: _run_smoke_training() ใน scripts/kaggle_train_pose_t5.py ทำการ hardcode smoke_args.epochs = 1 เสมอ ผ่านนิพจน์ที่ดูซับซ้อนแต่ return ค่าคงที่ ส่งผลให้ kernel ล้มเหลวภายใน 3 นาทีทุก version ตั้งแต่ v1 ถึง v39 (38 versions)
# ─── BUG (v1–v39) : epochs hardcoded = 1 เสมอ ──────────────────────────────
smoke_args.epochs = max(1, min(int(getattr(kaggle_args, "epochs", 1) or 1), 1))
#           ↑↑↑ min(x, 1) ≤ 1 เสมอ, max(1, ≤1) = 1 เสมอ
#
# seed อยู่ที่ epoch=76, target epoch=1 → 76 > 1 → trainer exit ทันที
# → zero new optimizer steps → noop-resume guard fires
# → RuntimeError: Resume completed with zero new optimizer steps
#    resume='/kaggle/working/phase1/_smoke/best_model_state.pt'
#    start_step=3075, final_step=3075

# ─── FIX (v40) : ให้ max_train_steps ควบคุมการหยุด smoke แทน ─────────────
smoke_args.epochs = 100000  # epoch ceiling ไม่มีทางถึง
# max_train_steps=20 (relative new steps จาก initial_step)
# → smoke รัน 20 steps จริง → Phase 1 เดินต่อ 3,725 steps → Phase 2 +375 steps
# ✓ v40 : chrF 96.77 / BLEU 97.73 / Exact 92%
เพิ่มเติม (belt-and-suspenders): notebook ยัง patch --smoke-steps 0 ใน Cell 3 และ Cell 4 เพื่อ skip smoke ทั้งหมดเป็น fallback เพิ่มเติม

14 บทสรุป

Curriculum Training v40 ประสบความสำเร็จในการ: (1) แก้ bug ที่ซ่อนอยู่ใน smoke training ที่ทำให้ล้มเหลวมา 38 versions (2) รัน pipeline สองเฟสจนจบใน ~57 นาทีบน Kaggle T4 GPU และ (3) ได้โมเดลที่ทำ chrF 96.77 ซึ่งสูงกว่า incumbent เดิม 9.82 คะแนน

ด้วย Exact Match 92% (23/25 ประโยค) โมเดลนี้แสดงให้เห็นว่า Curriculum Learning ที่ถูก implement อย่างถูกต้องสามารถช่วยให้โมเดลขนาดเล็ก (mT5-small) แปลภาษามือไทยได้อย่างแม่นยำสูงมาก แม้ว่าจะมีข้อมูล training เพียง 227 ตัวอย่างในเฟสสุดท้าย

ข้อผิดพลาดที่เหลืออยู่ (2/25) ล้วนเป็น pronoun/kinship confusion ซึ่งเป็นปัญหา inherent ของ low-resource scenario ที่ต้องการข้อมูล kinship signs เพิ่มเติม หรือ contrastive training สำหรับ near-synonym gestures

สถานะ Production: โมเดลถูก promote เข้า checkpoints/pose_t5_rtx4060_tsl51_only_export_verified/ และพร้อม serve ทันทีหลัง API restart · Kaggle dataset: orbitorls/thai-sign-tsl51-model
